Proyects

Pagina Web – QA Manual y Automation

Realicé During the study phase, we created a manual testing project following a Scrum methodology divided into sprints I designed the tests for the user stories following the BDD methodology, based on the defined acceptance criteria. Scenarios were created in Gherkin language to provide tests that are readable and understandable for the entire team, managing them with AssertThatto maintain traceability between requirements, tests, and results within Jira. The tests were executed per user story and the results were recorded in web and mobile environments.

During each sprint, functional tests were carried out and, at the end of it, exploratory tests were performed to identify defects not covered by the defined cases. Detected errors were reported in Jira, linking them to the corresponding user story to ensure proper traceability between requirements, tests, and defects. At the end of each sprint, the release was prepared.
